Context API in React JS

Context is used to reduce the drilling of data through the component tree by not passing it through every level of the tree. Context also uses states but make them more centralized. First, we need to create a context using React.createContext(default value), you can provide a default value or leave it blank. Creating context is the easiest step in the whole process. Now we need to create a Provider that will provide the consumer with all the variables, methods that it can consume. The provider is created using context, such as . We need to provide some values which will be provided to the consumer. Values can be objects, variables, or methods. Wrap the components inside the Provider component which are needed to consume data from it. Use Context.Consumer component inside the child function which needs the properties. These days we have seen almost every web app is having an option for a dark and light theme that users can select. We can use Context API to create an application that can switch between dark and light themes.
